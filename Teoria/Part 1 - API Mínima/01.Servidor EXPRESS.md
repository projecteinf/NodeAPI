#  Servidor Express

Aplicació que està escoltant peticions HTTP i que dóna una resposta quan algú accedeix a una ruta concreta.

Express és un framework molt utilitzat sobre Node.js per crear servidors web i APIs. 

La documentació oficial mostra que una aplicació mínima d'Express consisteix bàsicament en crear una instància d'Express, definir una ruta i posar el servidor a escoltar en un port.  [Express - Hello World example"](https://expressjs.com/en/starter/hello-world.html?utm_source=chatgpt.com)

---
# Què és un servidor web?

Un **servidor web** és un programa que queda en execució esperant peticions.

Per exemple, quan un navegador entra a:

```text
http://localhost:3000
```

el navegador està fent una petició HTTP a un servidor que escolta al port `3000`.

El servidor pot respondre amb diferents tipus de contingut:

```text
HTML
JSON
text pla
fitxers
errors
```

En el nostre cas, com que volem construir una API, normalment respondrem amb **JSON**.

---
# Què és Express?

**Express** és una llibreria/framework per a Node.js que facilita la creació de servidors HTTP.

Express ens dona una estructura senzilla per:

```text
definir rutes
respondre peticions
gestionar mètodes HTTP
afegir middleware
retornar JSON
gestionar errors
```

Exemple conceptual:

```ts
app.get("/", ...)
```

Això vol dir:

> quan arribi una petició GET a la ruta `/`, executa aquesta funció.

La pròpia documentació d'Express utilitza aquest patró per al seu exemple mínim “Hello World”.   [Express - Hello World example"](https://expressjs.com/en/starter/hello-world.html?utm_source=chatgpt.com)

# Què vol dir “aixecar un servidor”?

Quan diem que “aixequem un servidor”, volem dir que posem el programa en execució i el deixem **escoltant en un port**.

Exemple:

```ts
app.listen(3000, () => {
  console.log("Servidor escoltant al port 3000");
});
```

Això significa:

```text
El servidor queda actiu.
Espera peticions HTTP.
Escolta al port 3000.
Quan arriba una petició, Express mira si hi ha alguna ruta que encaixi.
```

# Estructura mínima d'un servidor Express amb TypeScript



Un primer fitxer podria ser:

```ts
import express from "express";

const app = express();
const port = 3000;

app.get("/", (req, res) => {
  res.send("Servidor Express funcionant");
});

app.listen(port, () => {
  console.log(`Servidor escoltant a http://localhost:${port}`);
});
```

Aquest codi té quatre parts importants.

---

## Importar Express

```ts
import express from "express";
```

Aquí importem Express per poder-lo utilitzar en el nostre projecte.

Express no forma part de Node.js per defecte. S'ha d'instal·lar com a dependència del projecte.

---

## Crear l'aplicació Express

```ts
const app = express();
```

Aquesta línia crea una aplicació Express.

La variable `app` representa el nostre servidor web. A partir d'aquesta variable podrem:

```text
definir rutes
afegir configuració
activar middleware
posar el servidor a escoltar
```

---

## Definir una ruta

```ts
app.get("/", (req, res) => {
  res.send("Servidor Express funcionant");
});
```

Aquesta línia defineix una ruta.

Vol dir:

```text
Quan algú faci una petició GET a la ruta /,
executa aquesta funció.
```

La ruta `/` és la ruta principal.

Per exemple:

```text
http://localhost:3000/
```

Dins la funció tenim dos objectes importants:

```ts
req
res
```

### `req`

Vol dir **request**, és a dir, la petició que arriba del client.

Conté informació com:

```text
la URL demanada
els paràmetres
el cos de la petició
les capçaleres HTTP
```

### `res`

Vol dir **response**, és a dir, la resposta que enviarem al client.

Amb `res` podem fer coses com:

```ts
res.send("Hola");
res.json({ missatge: "Hola" });
res.status(404).send("No trobat");
```

---

##  Posar el servidor a escoltar

```ts
app.listen(port, () => {
  console.log(`Servidor escoltant a http://localhost:${port}`);
});
```

Aquesta part arrenca el servidor.

El port és com una “porta d'entrada” del nostre ordinador. En desenvolupament és habitual utilitzar ports com:

```text
3000
3001
8080
```

Si fem servir el port `3000`, podrem provar el servidor obrint:

```text
http://localhost:3000
```

# Diferència entre Node.js i Express

Això és important perquè sovint l'alumnat ho barreja.

## Node.js

Node.js és l'entorn que permet executar JavaScript o TypeScript fora del navegador.

És a dir, permet fer programes de servidor.

## Express

Express és una llibreria que s'executa sobre Node.js i facilita la creació de servidors web.

Dit de manera simple:

```text
Node.js = motor
Express = eina per crear servidors més fàcilment
```

Sense Node.js, Express no pot funcionar.

# Per què fem servir TypeScript?

TypeScript ens permet escriure codi JavaScript amb tipus.

Per exemple:

```ts
const port: number = 3000;
```

Això ajuda a detectar errors abans d'executar el programa.

En projectes reals, TypeScript és molt útil perquè:

```text
redueix errors
millora l'autocompletat
fa el codi més mantenible
ajuda a entendre millor les dades
```


Instal·lació típica:

```bash

npm init -y
```

# 8. Exemple complet comentat

```ts
import express from "express";

// Creem una aplicació Express
const app = express();

// Definim el port on escoltarà el servidor
const port = 3000;

// Definim una ruta GET a l'arrel del servidor
app.get("/", (req, res) => {
  res.send("Servidor Express funcionant correctament");
});

// Posem el servidor en marxa
app.listen(port, () => {
  console.log(`Servidor escoltant a http://localhost:${port}`);
});
```

Aquest servidor només fa una cosa:

```text
Si entrem a http://localhost:3000
retorna el text "Servidor Express funcionant correctament".
```

Si intentem entrar a una ruta no definida, per exemple:

```text
http://localhost:3000/alumnes
```

Express retornarà un error de ruta no trobada, normalment un **404 Not Found**, perquè encara no hem definit aquest endpoint. La documentació oficial d'Express també indica aquest comportament en el seu exemple mínim. ([expressjs.com][1])
