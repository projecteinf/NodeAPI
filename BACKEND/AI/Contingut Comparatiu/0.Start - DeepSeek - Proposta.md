
# Building a Spotify‑Style Web API: From Zero to Professional

## Introduction

In this tutorial you will learn how to create a **web API** step by step.  
We start with a minimal **Express** server and gradually evolve it into a professional‑grade API using **TypeScript**, **Node.js**, and **PostgreSQL**.  

The theme is **Spotify** – you’ll build an API that manages users, playlists, tracks, and artists.  
By the end, you will have a solid foundation for building real‑world APIs.

**Technologies we will use:**

- Node.js + Express  
- TypeScript  
- PostgreSQL (you can adapt to MySQL/MariaDB with small changes)  
- Prisma (ORM – provides type safety and migrations)  
- JSON Web Tokens (JWT) for authentication  
- Zod for validation  
- dotenv, helmet, cors, express-rate-limit

---

## Stage 0 – Prerequisites & Setup

Before coding, make sure you have installed:

- **Node.js** (v18 or later)  
- **npm** (comes with Node)  
- **PostgreSQL** (or a cloud database like ElephantSQL) – [Download](https://www.postgresql.org/download/)  
- A REST client (Postman, Insomnia, or just `curl`)

```bash
mkdir spotify-api-tutorial
cd spotify-api-tutorial
```

## Stage 1 – Minimal Web API (No TypeScript)

Let’s build the smallest possible API – just a single endpoint that returns “Hello Spotify”.

1. Install Express:

```bash
npm install express
```

2. Create a file `server.js`:

```javascript
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send('Hello Spotify!');
});

app.listen(port, () => {
  console.log(`API running at http://localhost:${port}`);
});
```

3. Run it:

```bash
node server.js
```

Open `http://localhost:3000` – you’ll see the greeting.  
That’s a working (though trivial) web API.

## Stage 2 – Adding TypeScript & Basic Routing

Now we evolve to a more maintainable structure using TypeScript.

1. Install TypeScript and types:

```bash
npm install -D typescript @types/node @types/express ts-node nodemon
```

2. Create a `tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src/**/*"]
}
```

3. Move `server.js` to `src/index.ts` and rewrite it in TypeScript:

```typescript
import express, { Request, Response } from 'express';

const app = express();
const port = 3000;

app.use(express.json()); // to parse JSON bodies

app.get('/', (req: Request, res: Response) => {
  res.json({ message: 'Hello Spotify API (TypeScript)' });
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
```

4. Add a `dev` script to `package.json`:

```json
"scripts": {
  "dev": "nodemon --exec ts-node src/index.ts",
  "build": "tsc",
  "start": "node dist/index.js"
}
```

5. Run the dev server:

```bash
npm run dev
```

Now we have a TypeScript‑powered API. Let’s add a few routes that mimic Spotify resources:

```typescript
// Add after the root route
app.get('/tracks', (req: Request, res: Response) => {
  res.json([{ id: 1, name: 'Bohemian Rhapsody' }]);
});

app.get('/playlists', (req: Request, res: Response) => {
  res.json([{ id: 1, name: 'Rock Classics' }]);
});
```

## Stage 3 – Organising Code (MVC Pattern)

A professional API is not a monolithic file. We separate concerns:

- **Routes** – define endpoints  
- **Controllers** – handle request/response logic  
- **Services** – hold business logic  
- **Database** – (next stage)

Create the following folder structure inside `src/`:

```
src/
├── index.ts
├── routes/
│   └── trackRoutes.ts
├── controllers/
│   └── trackController.ts
├── services/
│   └── trackService.ts
└── types/
    └── index.ts
```

### Example: Track resource

**`src/types/index.ts`**

```typescript
export interface Track {
  id: number;
  name: string;
  duration: number; // in seconds
  artistId: number;
}
```

**`src/services/trackService.ts`** (in‑memory for now)

```typescript
import { Track } from '../types';

const tracks: Track[] = [
  { id: 1, name: 'Bohemian Rhapsody', duration: 354, artistId: 1 },
  { id: 2, name: 'Billie Jean', duration: 294, artistId: 2 },
];

export const getAllTracks = (): Track[] => tracks;

export const getTrackById = (id: number): Track | undefined => 
  tracks.find(track => track.id === id);
```

**`src/controllers/trackController.ts`**

```typescript
import { Request, Response } from 'express';
import * as trackService from '../services/trackService';

export const getAllTracks = (req: Request, res: Response) => {
  const tracks = trackService.getAllTracks();
  res.json(tracks);
};

export const getTrackById = (req: Request, res: Response) => {
  const id = parseInt(req.params.id);
  const track = trackService.getTrackById(id);
  if (!track) {
    return res.status(404).json({ error: 'Track not found' });
  }
  res.json(track);
};
```

**`src/routes/trackRoutes.ts`**

```typescript
import { Router } from 'express';
import { getAllTracks, getTrackById } from '../controllers/trackController';

const router = Router();

router.get('/', getAllTracks);
router.get('/:id', getTrackById);

export default router;
```

Finally, update `src/index.ts`:

```typescript
import express from 'express';
import trackRoutes from './routes/trackRoutes';

const app = express();
const port = 3000;

app.use(express.json());
app.use('/tracks', trackRoutes);

app.listen(port, () => console.log(`API running on port ${port}`));
```

Test `GET /tracks` and `GET /tracks/1` – they work, and the code is now modular.

## Stage 4 – Adding a Real Database (PostgreSQL + Prisma)

We need persistent storage. We’ll use **PostgreSQL** and **Prisma** (type‑safe ORM).

### 4.1 Install Prisma and the database driver

```bash
npm install @prisma/client
npm install -D prisma
```

### 4.2 Set up PostgreSQL

Create a database (e.g., `spotify_db`) and a user.  
Alternatively, use a connection URL like `postgresql://user:password@localhost:5432/spotify_db`.

### 4.3 Initialise Prisma

```bash
npx prisma init
```

This creates a `prisma/schema.prisma` file and a `.env` file.  
Edit `.env` with your database connection string:

```
DATABASE_URL="postgresql://your_user:your_password@localhost:5432/spotify_db"
```

### 4.4 Define the data model (Spotify theme)

Open `prisma/schema.prisma` and replace with:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String
  playlists Playlist[]
}

model Playlist {
  id        Int      @id @default(autoincrement())
  name      String
  userId    Int
  user      User     @relation(fields: [userId], references: [id])
  tracks    Track[]  @relation("PlaylistTracks")
}

model Artist {
  id      Int     @id @default(autoincrement())
  name    String
  tracks  Track[]
}

model Track {
  id        Int        @id @default(autoincrement())
  name      String
  duration  Int        // seconds
  artistId  Int
  artist    Artist     @relation(fields: [artistId], references: [id])
  playlists Playlist[] @relation("PlaylistTracks")
}
```

### 4.5 Run migrations

```bash
npx prisma migrate dev --name init
```

This creates the tables in your PostgreSQL database.

### 4.6 Generate Prisma Client

```bash
npx prisma generate
```

Now we can replace the in‑memory service with real database calls.

### 4.7 Update Track Service

Install `@prisma/client` (already installed) and use it.

**`src/services/trackService.ts`** (new version)

```typescript
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

export const getAllTracks = async () => {
  return await prisma.track.findMany({
    include: { artist: true } // include artist details
  });
};

export const getTrackById = async (id: number) => {
  return await prisma.track.findUnique({
    where: { id },
    include: { artist: true }
  });
};

export const createTrack = async (data: { name: string; duration: number; artistId: number }) => {
  return await prisma.track.create({ data });
};
```

### 4.8 Update the Controller to use `async/await`

**`src/controllers/trackController.ts`** (excerpt)

```typescript
export const getAllTracks = async (req: Request, res: Response) => {
  try {
    const tracks = await trackService.getAllTracks();
    res.json(tracks);
  } catch (error) {
    res.status(500).json({ error: 'Database error' });
  }
};

export const createTrack = async (req: Request, res: Response) => {
  try {
    const newTrack = await trackService.createTrack(req.body);
    res.status(201).json(newTrack);
  } catch (error) {
    res.status(400).json({ error: 'Invalid data' });
  }
};
```

Add a POST route in `trackRoutes.ts`:

```typescript
router.post('/', createTrack);
```

Now you can `POST /tracks` with a JSON body like:

```json
{
  "name": "Imagine",
  "duration": 183,
  "artistId": 1
}
```

and it will be saved to the database.

To make the API realistic, implement **Playlists** and **Artists** with full CRUD.

```typescript
        connect: { id: trackId }
      }
    }
  });

```

**Controller** and **route** follow the same pattern.

## Stage 6 – Validation & Error Handling

A professional API validates incoming data and returns consistent errors.

We’ll use **Zod** for validation.

### 6.1 Install Zod

```bash
npm install zod
```

### 6.2 Create validation schemas

**`src/validators/trackValidator.ts`**

```typescript
import { z } from 'zod';

export const createTrackSchema = z.object({
  name: z.string().min(1).max(100),
  duration: z.number().int().positive(),
  artistId: z.number().int().positive()
});

export const trackIdSchema = z.object({
  id: z.string().regex(/^\d+$/).transform(Number)
});
```

### 6.3 Middleware for validation

Create a generic validation middleware:

**`src/middleware/validate.ts`**

```typescript
import { Request, Response, NextFunction } from 'express';
import { AnyZodObject, ZodError } from 'zod';

export const validate = (schema: AnyZodObject) => 
  async (req: Request, res: Response, next: NextFunction) => {
    try {
      await schema.parseAsync({
        body: req.body,
        query: req.query,
        params: req.params
      });
      next();
    } catch (error) {
      if (error instanceof ZodError) {
        return res.status(400).json({ 
          error: 'Validation failed', 
          details: error.errors 
        });
      }
      next(error);
    }
  };
```

### 6.4 Use validation in routes

**`src/routes/trackRoutes.ts`**

```typescript
import { validate } from '../middleware/validate';
import { createTrackSchema } from '../validators/trackValidator';

router.post('/', validate(createTrackSchema), createTrack);
```

Now any invalid request (e.g., missing name) will return a clean 400 error.

### 6.5 Global error handler

Add a central error‑handling middleware at the end of `index.ts`:

```typescript
app.use((err: any, req: Request, res: Response, next: NextFunction) => {
  console.error(err);
  res.status(500).json({ error: 'Internal server error' });
});
```

## Stage 7 – Authentication (JWT)

Spotify‑style APIs require user authentication. We’ll implement **register**, **login**, and protect certain endpoints (e.g., creating a playlist should be tied to the logged‑in user).

### 7.1 Install dependencies

```bash
npm install bcryptjs jsonwebtoken
npm install -D @types/bcryptjs @types/jsonwebtoken
```

### 7.2 Create authentication service & controller

**`src/services/authService.ts`**

```typescript
import { PrismaClient } from '@prisma/client';
import bcrypt from 'bcryptjs';
import jwt from 'jsonwebtoken';

const prisma = new PrismaClient();
const JWT_SECRET = process.env.JWT_SECRET || 'supersecret';

export const registerUser = async (email: string, name: string, password: string) => {
  const hashedPassword = await bcrypt.hash(password, 10);
  return await prisma.user.create({
    data: { email, name, password: hashedPassword }
  });
};

export const loginUser = async (email: string, password: string) => {
  const user = await prisma.user.findUnique({ where: { email } });
  if (!user) throw new Error('Invalid credentials');
  const valid = await bcrypt.compare(password, user.password);
  if (!valid) throw new Error('Invalid credentials');
  const token = jwt.sign({ userId: user.id, email: user.email }, JWT_SECRET, { expiresIn: '1d' });
  return { user, token };
};
```

**Note:** Add `password` field to the `User` model in Prisma schema.  
Run `npx prisma migrate dev --name add_password` after updating the model.

### 7.3 Auth middleware to protect routes

**`src/middleware/auth.ts`**

```typescript
import { Request, Response, NextFunction } from 'express';
import jwt from 'jsonwebtoken';

export interface AuthRequest extends Request {
  userId?: number;
}

export const authenticate = (req: AuthRequest, res: Response, next: NextFunction) => {
  const authHeader = req.headers.authorization;
  if (!authHeader) return res.status(401).json({ error: 'No token provided' });
  const token = authHeader.split(' ')[1];
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!) as { userId: number };
    req.userId = decoded.userId;
    next();
  } catch {
    res.status(401).json({ error: 'Invalid token' });
  }
};
```

### 7.4 Protect playlist creation

When creating a playlist, the `userId` should be taken from the token, not from the request body.

**`src/controllers/playlistController.ts`** (partial)

```typescript
import { AuthRequest } from '../middleware/auth';

export const createPlaylist = async (req: AuthRequest, res: Response) => {
  const { name } = req.body;
  const userId = req.userId!;
  const playlist = await playlistService.createPlaylist(name, userId);
  res.status(201).json(playlist);
};
```

And in the route:

```typescript
router.post('/', authenticate, createPlaylist);
```

Now only authenticated users can create playlists.

## Stage 8 – Making It Professional

A production‑ready API needs more than just routes and authentication. Let’s add:

### 8.1 Environment variables

Create a `.env` file:

```
PORT=3000
DATABASE_URL=postgresql://...
JWT_SECRET=your_very_long_secret_key
```

Use `dotenv` to load them. Modify `index.ts`:

```typescript
import 'dotenv/config';
```

### 8.2 Security middleware

```bash
npm install helmet cors express-rate-limit
```

Add to `index.ts`:

```typescript
import helmet from 'helmet';
import cors from 'cors';
import rateLimit from 'express-rate-limit';

app.use(helmet());
app.use(cors());
const limiter = rateLimit({ windowMs: 15 * 60 * 1000, max: 100 });
app.use(limiter);
```

### 8.3 Structured logging

Install `pino` or use `morgan` for HTTP request logging.  
We’ll keep it simple with `morgan`:

```bash
npm install morgan
npm install -D @types/morgan
```

```typescript
import morgan from 'morgan';
app.use(morgan('combined'));
```

### 8.4 Configuration management

Create `src/config/index.ts`:

```typescript
export const config = {
  port: process.env.PORT || 3000,
  jwtSecret: process.env.JWT_SECRET!,
  databaseUrl: process.env.DATABASE_URL!
};
```

Use `config.port` instead of hardcoded values.

### 8.5 Separate Prisma client instance

Create `src/lib/prisma.ts`:

```typescript
import { PrismaClient } from '@prisma/client';
export const prisma = new PrismaClient();
```

Then import `prisma` everywhere instead of creating multiple instances.
