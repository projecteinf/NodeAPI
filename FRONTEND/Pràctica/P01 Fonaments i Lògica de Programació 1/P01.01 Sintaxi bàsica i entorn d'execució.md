# Pràctica: "Simulador de Login"

**Objectiu:** Crear el sistema lògic de validació d'un usuari i simulació de subscripció per a la nostra aplicació de música, utilitzant només la consola del navegador.
## El banc de proves (Consola)

1. Obre el teu navegador, obre una pestanya buida (`about:blank`) i accedeix a la **Consola (F12)**.

2. Neteja la consola fent clic a la icona del senyal de prohibit o prement `Ctrl + L`.
## Definició de les variables (constants)

Copia i enganxa el següent bloc de codi a la teva consola i prem `Enter`. Aquestes seran les nostres dades mestres emmagatzemades:

```javascript
const USUARI_CORRECTE = "profe_web";
const CONTRASENYA_CORRECTE = "12345";
const EDAT_MINIMA = 14;
```
## Programació de la lògica de validació

Ara has de redactar, a continuació a la mateixa consola, un script que simuli un intent d'accés.

Pots escriure en línies diferents sense que s'executi prement `Shift + Enter` a cada salt de línia. El codi ha de complir les següents condicions:

1. Crea una variable per a l'usuari que intenta entrar (`usuariIntent`) i una altra per a la contrasenya (`passIntent`). Posa-hi els valors que vulguis per fer proves.

2. Crea una variable per a l'edat de l'usuari que s'està connectant (`let edatIntent`), però assegura't que el valor sigui un **Number** (si el reculls com a text, força la seva conversió).

3. Utilitza una estructura `if / else` per verificar:
	* Si l'usuari i la contrasenya coincideixen amb les constants correctes.
	
	* Si les dades són correctes, s'ha de comprovar l'edat: si és major o igual a `EDAT_MINIMA`, mostrarà per consola: *"Benvingut a Spotify de classe. Carregant la teva música..."*. Si és menor, mostrarà: *"Error: Ets massa jove per utilitzar aquesta aplicació"*.
	
	* Si l'usuari o la contrasenya són incorrectes, ha de mostrar: *"Credencials incorrectes. Accés denegat."*
## Viatjar en el temps amb el Depurador

- Després de la definició de les variables afegeix la sentència `debugger;` per a aturar l'execució de codi.

- Quan s'executa la comanda `debugger` el navegador salta automàticament a la pestanya Sources (Fonts) i el codi es queda congelat subratllat en groc sobre la paraula `debugger`.

- Passa el ratolí per sobre les variables per a obtenir-ne el seu valor.

- Modificar el valor de passIntent

- Prem el botó de "Play" blau (Resume script execution) o la tecla F8.

Pregunta per a la memòria: Què ha passat? El programa t'ha denegat l'accés o t'ha deixat entrar? Explica com canviar el valor de la variable a través del depurador ha canviat el destí de l'execució del codi.
## Tasca 4: Comprovació de bucles (La reproducció inicial)

Un cop l'usuari hagi "entrat" correctament (simula-ho posant les credencials bones), afegeix al final del codi un bucle `for` que simuli la càrrega inicial dels teus 3 gèneres musicals preferits, mostrant a la consola missatges com:

```text
Configurant gènere musical favorit número 1...
Configurant gènere musical favorit número 2...
Configurant gènere musical favorit número 3...
```
