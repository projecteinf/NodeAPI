# Pràctica

## Exploració de l'Entorn Client i Investigació del Sandbox

Investigació pràctica de les capacitats, limitacions i seguretat del navegador.
### Objectius de la pràctica:

1. Comprendre el comportament del navegador com a entorn d'execució tancat (Sandbox).

2. Experimentar amb el bloqueig de seguretat SOP i entendre els errors de CORS que sorgiran en connectar la interfície amb l'API de Spotify.

3. Utilitzar les Eines de Desenvolupador (DevTools) per analitzar el comportament del client.

### Exploració de les capacitats del client mitjançant l'objecte `navigator`.

L'objecte de JavaScript `navigator` conté tota la informació sobre el navegador de l'usuari.

1. Obriu qualsevol pàgina web buida al vostre navegador (per exemple, `about:blank`).

2. Premeu `F12` (o clic dret -> *Inspeccionar*) i aneu a la pestanya **Console (Consola)**.

3. Escriviu les següents ordres i observeu/anoteu els resultats per incloure'ls a la memòria:
```javascript
console.log(navigator.userAgent);
console.log(navigator.language);
console.log(navigator.onLine);
```

4. **Pregunta per a la memòria:** Què és el `userAgent` i per què creus que és útil que el codi de client sàpiga des de quin navegador i sistema operatiu s'està executant?

### Demostració de les limitacions del Sandbox (Seguretat de fitxers local)

Anem a comprovar com el navegador ens impedeix accedir lliurement a fitxers interns de l'ordinador per seguretat.

1. Creeu una carpeta qualsevol al vostre ordinador. Dins d'aquesta, creeu dos fitxers:
	* Un fitxer de text anomenat `dades_secretes.txt` que contingui la frase: *"Això és un fitxer privat del sistema operatiu del client."*
	* Un fitxer HTML anomenat `index.html`.

2. Dins de l'HTML, afegiu un script bàsic que intenti llegir de forma automàtica el fitxer de text utilitzant la comanda de xarxa `fetch`:
```html
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Prova Sandbox</title>
</head>
<body>
    <h1>Prova de seguretat del client</h1>
    <script>
    // 1. Intentem "anar a buscar" el fitxer local utilitzant el motor de xarxa del navegador
    fetch('dades_secretes.txt') 
        
        // 2. Si el navegador ens deixés agafar-lo, el transformaríem en text llegible
        .then(response => response.text()) 
        
        // 3. Si tot hagués anat bé, es mostraria el contingut del text a la consola
        .then(data => console.log("Contingut llegit amb èxit:", data)) 
        
        // 4. Si el sistema de seguretat (Sandbox) en bloca el pas, capturem l'error i el mostrem
        .catch(error => console.error("El Sandbox ha bloquejat la petició. Error:", error));
    </script>
</body>
</html>
```
3. Obriu el fitxer `index.html` fent-hi doble clic des del vostre escriptori (veureu que a la barra d'adreces de la URL utilitza el protocol  `file://...` ).

4. Obriu la **Consola (F12)**. Què ha passat? Quin error es mostra?

5. **Exercici de reflexió:** Expliqueu per què el navegador actua d'aquesta manera bloquejant el fetch quan treballem amb el protocol file://. Quina diferència hi ha entre demanar un fitxer a un servidor web extern (mitjançant http://) i intentar que el codi JavaScript accedeixi directament a una ruta del teu disc dur (mitjançant file://)? Què passaria amb la teva privacitat si el Sandbox no bloqués el protocol file:// a les comandes de xarxa?.
### Provocació simulada d'un error de CORS

Quan comencem a programar en React la interfície de Spotify, l'error de CORS serà el vostre pitjor enemic si el vostre backend no està ben configurat. Anem a provocar-lo voluntàriament per aprendre a identificar-lo.

1. Aneu a qualsevol lloc web comercial gran (per exemple, `https://www.google.com` o `https://www.wikipedia.org`).

2. Obriu les Eines de Desenvolupador (**Consola**).

3. Executareu una ordre `fetch` des d'aquella consola per intentar demanar dades a una API pública que té restringit l'accés des d'altres dominis que no siguin els seus de producció, o simplement des d'un domini diferent de l'original. Escriviu això a la consola:
```javascript
fetch('https://www.apple.com');
```

4. Observeu l'error que apareix instantàniament a la consola. Tindrà un aspecte similar a: *"Access to fetch at... from origin... has been blocked by CORS policy..."*

5. Captureu la pantalla de l'error.
