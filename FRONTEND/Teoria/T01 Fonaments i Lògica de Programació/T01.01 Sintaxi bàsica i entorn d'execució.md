# Introducció a l'Entorn Client i Sintaxi Fonamental de JavaScript

## On estem programant? El model Client/Servidor (Introducció)

Quan naveguem per Internet, sempre hi ha dos ordinadors que parlen entre ells:

* **El Servidor (Backend):** És un ordinador remot on es guarden les dades massives (per exemple, la base de dades amb els milions de cançons de Spotify). El codi del servidor processa aquestes dades de forma oculta per a l'usuari.

* **El Client (Frontend):** És el teu propi dispositiu (ordinador, mòbil). Executa el **navegador web**, que rep les instruccions i s'encarrega de pintar la pantalla i fer-la interactiva.

En aquest mòdul aprendrem a programar el **Client**, utilitzant l'únic llenguatge que tots els navegadors del món entenen de forma nativa: **JavaScript**.
## El nostre laboratori: La Consola del Navegador

Per començar a programar no necessitem instal·lar entorns complexos. El teu propi navegador (Chrome, Firefox, Edge) té integrat un entorn d'execució de JavaScript.

* **Com s'hi accedeix?** Obre una pestanya buida al navegador, premeu la tecla `F12` (o fes clic dret $\rightarrow$ *Inspeccionar*) i selecciona la pestanya **Console (Consola)**.

* Qualsevol línia de codi JavaScript que escriguis aquí s'executarà a l'acte utilitzant la memòria i el processador del teu ordinador.
### El Debugger (depuració)

Quan programem, els ordinadors fan milers d'operacions per segon i de vegades és difícil entendre per què el nostre codi no fa el que volem. En lloc d'endevinar-ho, podem utilitzar la paraula clau debugger; dins del nostre codi.

Quan el navegador llegeix la paraula debugger;, atura el temps i congela l'execució del programa exactament en aquesta línia. En aquell moment, la pantalla s'enfosquirà i podràs:

    Passar el ratolí per sobre de qualsevol variable per veure quin valor té en aquell instant precís.

    Avançar el codi línia per línia fent clic a la fletxa de pas de les DevTools (a la pestanya Sources o Fonts).

    Modificar a la consola el valor de les variables "al vol" per veure com canvia el comportament del programa sense haver de tornar a escriure tot el codi.
    
## Sintaxi Fonamental: Guardar dades (Variables i Àmbits)

En el JavaScript modern (ES6+), tenim dues maneres principals de crear variables:

* **`let`:** S'utilitza per a valors que **poden canviar** al llarg del temps (per exemple, els minuts reproduïts d'una cançó).

* **`const`:** S'utilitza per a valors **constants** que mai canviaran (per exemple, el nom d'usuari un cop ha iniciat sessió).

> ⚠️ **Nota històrica:** Antigament es feia servir la paraula `var`. Avui dia **està totalment desaconsellada** perquè no respecta l'àmbit de bloc (crea problemes de visibilitat de les dades fora de lloc), provocant errors difícils de trobar. Sempre farem servir `let` o `const`.

```javascript
const usuari = "Joan89"; // Aquest valor no canviarà
let cancoActual = "Despechá"; // Podem canviar-lo més endavant
cancoActual = "Motomami"; // Canvi de valor correcte
```
## Tipus de dades i Conversions

JavaScript és un llenguatge **dinàmic**, cosa que vol dir que no cal dir-li quin tipus de dada és cada variable; ell ho dedueix directament:

* **String (Text):** `"Rosalía"`, `'Bad Bunny'` (va entre cometes).

* **Number (Número):** `180` (durada en segons), `4.5` (els decimals es posen amb punt).

* **Boolean (Booleà):** `true` o `false` (cert o fals; útil per saber, per exemple, si la música està en *pausa*).
### Conversions de tipus (Coerció)

A vegades, JavaScript intenta ser "massa intel·ligent" i converteix dades de forma automàtica, fet que pot provocar errors si no es va amb compte:

```javascript
let a = "5"; // És un text (String)
let b = 2;   // És un número (Number)
console.log(a + b); // El resultat serà "52" (els enganxa), no 7!
```

Per evitar-ho, podem forçar nosaltres la conversió manual:

```javascript
let aNum = Number("5"); // Ara ja és el número 5
console.log(aNum + b);   // El resultat ara sí que és 7
```
## Estructures de Control: Decisions i Bucles

### Prendre decisions (`if` / `else`)

Permet executar un bloc de codi només si es compleix una condició:

```javascript
let edatUsuari = 16;

if (edatUsuari >= 18) {
    console.log("Accés permès a la versió Premium.");
} else {
    console.log("Accés limitat (Versió per a menors).");
}
```
### Repetir accions (El bucle `for`)

Imagina que vols llistar les cançons d'una llista de reproducció. En lloc d'escriure el codi deu vegades, utilitzem un bucle per repetir l'acció de forma controlada:

```javascript
// Aquest bucle s'executarà 5 vegades (mentre la i vagi de 0 a 4)
for (let i = 0; i < 5; i++) {
    console.log("Reproduint cançó número: " + i);
}
```
