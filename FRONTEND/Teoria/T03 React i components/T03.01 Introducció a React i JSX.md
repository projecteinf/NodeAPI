# Filosofia de Components i Sintaxi JSX

## Per què React? El problema del DOM natiu

Fins ara, si volíem canviar el títol d'una cançó a la pantalla, havíem de fer tres passos:

1. Buscar l'element amb `querySelector`.
2. Modificar la variable en memòria.
3. Actualitzar el text amb `.textContent`.

En aplicacions reals amb milers de dades, sincronitzar la memòria del programa amb el que veu l'usuari genera el que s'anomena "fragmentació de l'estat" (errors on el codi diu una cosa i la pantalla en mostra una altra).

React capgira el procés: tu modifiques les dades, i un motor intel·ligent s'encarrega d'actualitzar el DOM per tu d'una manera ultraeficient gràcies al Virtual DOM (una còpia lleugera de l'arbre a la memòria RAM). **Tu només te n'encarregues de gestionar les dades (l'Estat), i React s'encarrega de pintar la pantalla automàticament cada vegada que aquestes dades canvien.**
## La "Cuina" de React: Quines llibreries necessitem?

Perquè una pàgina web buida es converteixi en una aplicació de React, el navegador necessita descarregar tres eines (o llibreries) diferents. En aquesta unitat les descarregarem per CDN (enllaços directes a servidors d'Internet) per veure-les de forma aïllada:

- react.js (El nucli): És el cervell. Conté la lògica per gestionar els components, el cicle de vida de l'aplicació i l'estat de les dades. No sap res de pantalles ni de navegadors; només gestiona la lògica pura.

- react-dom.js (El traductor visual): És el braç executor. Agafa les ordres del nucli de React i les tradueix en ordres reals per al navegador (appendChild, etc.). És la llibreria que connecta el cervell de React amb la pantalla de l'usuari.

- babel.js (El compilador/traductor de codi): Els navegadors no entenen React ni JSX de forma nativa. Si escrius HTML dins de JavaScript, el navegador donarà un error de sintaxi. Babel actua com un traductor en temps real: llegeix el nostre codi modern i el transforma en JavaScript clàssic que qualsevol navegador pugui entendre.

## Com funciona l'entorn realment? (El cicle de traducció)

Quan preparem el nostre fitxer HTML, fixat en com es connecten aquestes peces:
```HTML
	<div id="root"></div>
	
	<script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
	
	<script type="text/babel">
	    // El nostre codi React anirà aquí dins
	</script>
```
### Què passa per darrere quan obrim la pàgina?

1. El navegador llegeix el script type="text/babel". Com que no és un script normal, no l'executa.

2. Babel agafa el control, llegeix tot el text que hem escrit, detecta les etiquetes HTML barrejades i les tradueix a funcions pures de JavaScript (com `React.createElement()`).

3. Un cop transformat, React-DOM agafa aquest codi netejat i l'enganxa (render) dins del cubell buit anomenat `<div id="root">`.

## Hello World, en React

Nom fitxer: `indexReact.html`
```HTML
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>El meu primer React</title>
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
</head>
<body>

    <div id="root"></div>

    <script type="text/babel">
    
        // A. Busquem el cubell físic de l'HTML
        const cubellFisic = document.querySelector('#root');
        // En els exemples, normalment veureu utilitzar getElementById('root');
        
        // B. Creem l'arrel virtual de React sobre aquest cubell
        const arrelReact = ReactDOM.createRoot(cubellFisic);
        
        // C. Li demanem a React que pinti (render) un text utilitzant JSX 
        arrelReact.render(<h1>Hola Món des de React!</h1>);

    </script>
</body>
</html>
```
## Què és un Component?

A React, una pàgina web ja no és un fitxer HTML llarg de 1.000 línies. Una aplicació és un trencaclosques fet de petites peces independents i reutilitzables anomenades **Components**.

A nivell de codi, un component de React és, literalment, **una funció `arrow` de JavaScript** que retorna el codi visual que s'ha de pintar a la pantalla. El nom d'aquestes funcions **sempre ha de començar en majúscula**.

```javascript
// El nostre primer component de React!
const ElMeuReproductor = () => {
    return <h1 className="verd">El nostre Spotify</h1>;
};

// Renderitzem el component
arrelReact.render(<ElMeuReproductor />);
```

## La sintaxi JSX (JavaScript + XML)

En el codi anterior hem escrit una etiqueta `<h1>` directament dins del JavaScript, sense cometes! Això no és HTML, es diu **JSX**.

JSX és una extensió de sintaxi que ens permet escriure l'estructura visual de la nostra aplicació de forma idèntica a l'HTML, però amb tot el poder de JavaScript integrat.

### Com combinem dades i JSX? (Les claus `{}`)

Si volem que un component mostri una variable de JavaScript, només l'hem d'encloure dins de **claus `{}`**. Qualsevol cosa que vagi dins de claus s'avaluarà com a codi actiu:

```javascript
const TargetaCanço = () => {
    const titol = "Milionària";
    const artista = "Rosalía";

    // Fusionem el text i la lògica dins del JSX fent servir les claus
    return (
        <div className="targeta">
            <h3>{titol.toUpperCase()}</h3>
            <p>Artista: {artista}</p>
        </div>
    );
};

```

> ⚠️ **Diferències clau amb HTML natiu:**
> * Com que la paraula `class` ja està reservada en JavaScript per a les classes d'objectes, en JSX per posar una classe de CSS hem d'escriure obligatòriament **`className`**.
> * Totes les etiquetes s'han de tancar obligatòriament (per exemple, un `<img>` de HTML ara s'ha d'escriure `<img />`).
## Renderitzar llistes en React (El retorn del `.map()`)

A React, `.map()` és l'eina oficial per transformar llistes de dades en llistes de targetes visuals:

```javascript
const LlistaMusical = () => {
    const cancons = ["Despechá", "Candy", "Monaco"];

    return (
        <ul>
            {cancons.map((titol, index) => {
                // Transformem cada text en una etiqueta <li> de JSX
                return <li key={index}>{titol}</li>;
            })}
        </ul>
    );
};
```

*Nota:* Quan pintem llistes a React, hem d'afegir l'atribut **`key`** a l'element pare de la llista perquè React sàpiga identificar cada peça de forma única a la memòria.
