# DOM (Document Object Model) 

## Què és el DOM 

```html
<div id="music-player" class="player-dark-theme">
    
    <div class="song-info">
        <h2 class="current-title">Despechá</h2>
        <p class="current-artist">Rosalía</p>
    </div>

    <div class="player-controls">
        <button class="btn-control btn-prev">Anterior</button>
        
        <button class="btn-control btn-play">▶ Reproduir Cançó</button> 
        
        <button class="btn-control btn-next">Següent</button>
    </div>

</div>

<div class="seccio-recomanacions">
    <h3>També et pot agradar:</h3>
    <button class="btn-control btn-play">▶ Reproduir Ràdio</button> 
</div>
```

Quan el navegador descarrega un fitxer HTML d'Internet, no es limita a llegir el text. Internament, el motor de renderitzat transforma tot l'HTML en una estructura de l'estil "arbre genealògic" on cada etiqueta (un `<div>`, un `<h1>` o un `<p>`) és un **node** o un **objecte** independent.

```text
document (Arrel de tot)
 └── body
      ├── div#music-player (Contenidor principal)
      │    ├── div.song-info
      │    │    ├── h2.current-title
      │    │    └── p.current-artist
      │    └── div.player-controls
      │         ├── button.btn-prev
      │         ├── button.btn-play  
      │         └── button.btn-next
      └── div.seccio-recomanacions
           ├── h3
           └── button.btn-play       
```

Aquesta estructura es coneix com a **DOM (Document Object Model)**.

JavaScript té la capacitat de connectar-se a aquest arbre mitjançant l'objecte global **`document`**. A través d'ell, podem buscar qualsevol element de la pantalla, canviar-li el text, canviar-li el color CSS o, fins i tot, crear etiquetes noves des de zero.
## Els Selectors

Abans de modificar un element de la pàgina web, l'hem de trobar. El JavaScript modern utilitza dues eines principals que funcionen exactament igual que els selectors de CSS:

* **`document.querySelector('selector')`:** Busca i retorna el **primer** element que coincideixi amb el selector de CSS (pot ser una etiqueta, una classe `.classe` o un identificador `#id`).
* **`document.querySelectorAll('selector')`:** Busca i retorna **tots** els elements que coincideixin, guardant-los dins d'una llista (un array de nodes).

```javascript
// Busquem el reproductor de música per la seva ID de HTML
const reproductor = document.querySelector('#music-player'); // primer div de body

// Busquem tots els botons que tinguin la classe ".btn-play"
const botonsPlay = document.querySelectorAll('.btn-play'); // 2 botons
```
## Modificar el contingut i els estils

Un cop tenim l'element capturat en una variable, podem alterar les seves propietats:

* **`element.textContent`:** Permet llegir o canviar el text que hi ha dins de l'etiqueta (de forma segura, ja que només injecta text pla).
* **`element.classList`:** Permet afegir (`.add()`) o treure (`.remove()`) classes de CSS dinàmicament per canviar l'aspecte visual.

```javascript
const titolCançó = document.querySelector('.current-title');
titolCançó.textContent = "Milionària - Rosalía"; // El text canvia a la pantalla de l'usuari!

titolCançó.classList.add('text-green'); // Li apliquem un estil CSS dinàmic
```
## Creació dinàmica d'elements (Bones Praxis)

Imagina que el nostre backend ens envia una llista de 10 cançons noves. No anem a escriure l'HTML de les 10 targetes a mà. Hem de fer que JavaScript les crei automàticament.

La manera més neta i segura de fer-ho és utilitzant:

* **`document.createElement('etiqueta')`:** Crea una etiqueta buida a la memòria del navegador.
* **`pare.appendChild(fill)`:** Enganxa el nou element dins d'un element pare existent a la pantalla.

```javascript
// Creem un element <li> buit a la memòria i li associem la classe titol
const novaCancoLi = document.createElement('li');
novaCancoLi.classList.add('titol');  

// Li posem el text de la cançó
novaCancoLi.textContent = "Candy - Rosalía";

// Al contenidor de la llista (<ul>) li injectem el <li> a dins
const llistaUl = document.querySelector('#playlist-container');
llistaUl.appendChild(novaCancoLi); // Ara ja és visible a la pàgina!
```

### Tasca 2: La funció pura de creació visual (Bones Praxis)

Dins de l'etiqueta `<script>`, has de programar una funció de bona praxi.

1. Crea una funció fletxa anomenada `crearTargetaHtml`.
2. Ha de ser una **funció pura**: ha de rebre com a paràmetre un únic **objecte cançó** (per exemple, `{titol: "...", artista: "...", ...}`).
3. La funció s'ha de encarregar de crear un element a la memòria: un `<div>` que farà de contenidor de la targeta. Li ha d'afegir la classe CSS `targeta-canco`.
4. Dins d'aquest `<div>`, ha de crear i introduir un `<h3>` amb el títol de la cançó i un `<p>` amb el nom de l'artista.
5. Finalment, la funció **ha de retornar** el `<div>` resultant (encara no l'ha d'enganxar a la pantalla).

### Tasca 3: Pintar el catàleg sencer (Ús de `.map()` o `.forEach()`)

1. Crea una funció anomenada `renderitzarCatatleg`.
2. Ha de ser una **funció pura**: rebrà com a paràmetres la llista d'objectes a pintar (array) i el contenidor destí del HTML on s'han d'enganxar.
3. Dins de la funció, captura el contenidor destí fent servir `document.querySelector()`.
4. Recorre l'array rebut. Per a cada cançó, crida la funció de la Tasca 2 (`crearTargetaHtml`) per obtenir el seu node i enganxa'l dins del contenidor destí utilitzant `.appendChild()`.
5. Executa la funció pasant-li `baseDeDadesMusica` i el selector del contenidor principal. En desar i obrir el fitxer HTML al navegador, les 4 targetes s'haurien de dibuixar en color gris i verd automàticament.

### 🔑 SOLUCIONARI DE LA PRÀCTICA 3 (Per al docent)

Aquest és el bloc de JavaScript que l'alumne hauria de desenvolupar dins del seu script:

```javascript
// Tasca 2: Funció pura que genera un element del DOM aïllat
const crearTargetaHtml = (canco) => {
    // El debugger s'aturarà aquí a cada cançó, tal com demana la Tasca 4
    debugger; 

    // 1. Creem el contenidor principal de la targeta
    const targetaDiv = document.createElement('div');
    targetaDiv.classList.add('targeta-canco');

    // 2. Creem l'etiqueta per al títol
    const titolH3 = document.createElement('h3');
    titolH3.textContent = canco.titol;

    // 3. Creem l'etiqueta per a l'artista
    const artistaP = document.createElement('p');
    artistaP.textContent = "Artista: " + canco.artista;
    
    // 4. Creem l'etiqueta per al gènere
    const genereP = document.createElement('p');
    genereP.textContent = "Gènere: " + canco.genere;

    // 5. Ajuntem l'estructura (fiquem els fills dins del pare)
    targetaDiv.appendChild(titolH3);
    targetaDiv.appendChild(artistaP);
    targetaDiv.appendChild(genereP);

    // 6. Retornem el bloc sencer construït a memòria
    return targetaDiv;
};

// Tasca 3: Funció pura encarregada d'injectar les dades a la pantalla
const renderitzarCatatleg = (llistat, selectorDesti) => {
    const contenidor = document.querySelector(selectorDesti);
    
    // Bones praxis: Netegem el contenidor per si de cas hi havia dades prèvies
    contenidor.textContent = "";

    // Recorrem l'array element per element
    llistat.forEach(elementMusical => {
        // Generem la targeta gràfica cridant la funció pura
        const targetaGrafica = crearTargetaHtml(elementMusical);
        // L'injectem realment al DOM de la pàgina
        contenidor.appendChild(targetaGrafica);
    });
};

// Execució final per activar la renderització en obrir la pàgina
renderitzarCatatleg(baseDeDadesMusica, '#contenedor-principal');

```
